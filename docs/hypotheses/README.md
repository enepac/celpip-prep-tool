# Phase Hypotheses

One file per phase. Written at the start of the phase, before any work begins on that phase.

## Filename format

```
phase-N-hypotheses.md
```

Example: `phase-1-hypotheses.md`

## File structure

```markdown
# Phase N Hypotheses

Phase: [name and dates]
Written: [date]

## Hypothesis 1: [short title]

**Claim:** [One sentence stating what you believe is true.]

**Why you believe it:** [One or two sentences of reasoning.]

**Measurement method:** [How will you know if this is true or false? Be specific — what data, from whom, collected how?]

**Falsification threshold:** [The specific outcome that would prove this hypothesis wrong. E.g., "If fewer than 3 of 10 beta users complete a full scoring session, this hypothesis is falsified."]

**Status:** [Pending / Confirmed / Falsified — updated during the phase]

**Outcome:** [What actually happened — filled in at phase closeout]

---

[Repeat for Hypothesis 2 and 3. Keep to 1–3 hypotheses per phase. More than 3 is unfocused.]
```

## Why this format

Hypotheses without falsification thresholds are opinions. Writing the threshold before the phase starts prevents confirmation bias: you cannot retroactively define what "success" meant. The measurement method column forces specificity — "users will like it" is not a hypothesis.

Keep to 1–3 per phase. The goal is to make one or two genuinely important bets per phase, not to enumerate everything that might be learned.
